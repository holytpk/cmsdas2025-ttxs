    Customizing your analysis

    There are two main operations that can be performed in the Pepper processor: defining variables and adding cuts. In this section, you will do both of these things.

    Introduction: Defining variables

    We have seen that the main script for processing events from NanoAOD is processor_ttxs_dilep.py. This contains the analysis procedure for our measurement, but inherits methods from two broader processor classes:

    pepper/processor_basic.py (generic methods) -> pepper/processor_ttbarll.py (dilepton channel utilities) -> processor_ttxs.py

    Let's start with our measurement's processor, processor_ttxs_dilep.py. Have a look at the main methods process_selection and process_selection_jet_part. These contain the workflow for the data analysis: when to cut, apply scale factors, etc. You will see several instances of function calls like this:

    selector.set_column("nbtag", self.num_btags)

    This defines a new variable - in this case nbtag - by giving a reference to a function that can compute this variable from already existing branches and variables.

    Let us look at the function num_btags in question:

    def num_btags(self, data):
    jets = data["Jet"]
    nbtags = ak.sum(jets["btagged"], axis=1)
    return nbtags

    The function takes an already existing variable, Jet from the data, and calculates the number of b-tagged jets by summing over the btagged field of the jet (which is 0 for non-b-tagged and 1 for b-tagged jets). It then returns the calculated quantity. Remember that all these operations are done in a columnar way!

    Going back to the main processor, after calling selector.set_column, the nbtag field will be added to the data and can be accessed in all following variable definitions and cuts, as well as in histogram filling.

    Sometimes the functions for variable definitions are more complicated and have additional arguments. They can be passed from the main processor using partial, like e.g. for the Lepton variable:

    selector.set_column("Lepton", partial(
    self.build_lepton_column, is_mc, selector.rng))

    Finally, there is also the possibility to compute several variables at the same time with selector.set_multiple_columns.

    Defining your own variable

    You might have noticed that the plots you made in Exercise 1 do not look very good in the ee and μμ channels - they are full of Z+jets background. We would like to correct this now by looking at a variable that separates tt¯ and Z+jets: the invariant lepton mass mℓℓ.

    To-do

    Define a new variable mll in the Pepper processor!

    You can follow these rough steps:

    • Create a new function in processor_ttxs_dilep.py with the same parameters as the num_btags example.
    • Calculate the invariant mass of the two leptons in this function. You can get information on the leptons from the Lepton column. Remember that this is a 2-dimensional array!
    • In the function, return the invariant mass that you calculated.
    • Add a call to selector.set_column in the process_selection method. Think about where this call should go - which cut is required to be passed so that the invariant lepton mass is well-defined?
    • Add a new histogram for mll to your config, similar as in Exercise 1.

    If you are done, run in debug mode to check if your code works. If it does, feel free to run over the full dataset and make a plot of your new variable!

    Adding cuts

    Adding cuts is very similar to defining new variables. Take, as an example, the cut requiring a certain number of jets:

    selector.add_cut("Has jet(s)", self.has_jets)

    The cut has a name (Has jet(s)) that it will be called in the cutflow and histogram, and is defined via the function has_jets. It can be found in pepper/processor_basic.py:

    def has_jets(self, data):
    """Require events with minimum number of jets."""
    return self.config["num_jets_atleast"] <= ak.num(data["Jet"])

    As you can see, it works similar to a function defining a variable - instead of returning an array of numbers, it returns a boolean expression (an array mask) determining which events to keep and which ones to cut away. The value at which the number of jets is cuts is read from the config (doing it this way is not required, however).

    With this, you should be ready to add cuts on your newly defined variable mll!

    To-do

    Add a cut requiring a mll value of at least 20 GeV for each event. The purpose of this cut is to remove events where the lepton pair is non-prompt, e.g. resulting from lepton pair production by a radiated photon.

    You can add the value of 20 GeV to the config and read it from the function defining the cut, or add it directly to the code for now.

    Run in debug mode to check whether it worked.

    To-do

    Add a second cut removing the Z boson peak in mll, by removing events for which |mℓℓ−mZ| is less than 15 GeV (mZ=91 GeV is the Z boson mass).

    Apply this cut only to same-flavor events (i.e. ee and μμ) - otherwise you pointlessly throw away statistics in the eμ channel. To do so, you can use the categories keyword argument of the add_cut function - look at the documentation of add_cut in pepper/selector.py to try to find out how it works!

    Again run in debug mode. If you are happy, you can run on the full dataset and compare the plots with what you had before.

    Optional: More variables

    If you have enough time and feel like it, you can try adding and plotting more variables. A non-exhaustive list is (choose the ones that you think are most interesting):

    • the pT of the dilepton system;
    • the scalar sum HT of all hadronic activity (in terms of jet transverse momenta) in the event;
    • the invariant mass mℓb of the leading lepton and the leading b jet;
    • the invariant mass mℓℓbb of both leptons and the two leading b jets (think about in which category this variable makes sense);
    • the invariant mass mℓbmin of one lepton and one b jet, where both are chosen from all possible leptons and jets such that the difference |mℓbmin−mtop| is minimized.
